# akiya-watch 恒久ルール

このプロジェクトに関わるすべての作業で、以下を厳守する。

## URL マスタの扱い
- `urls.yaml` が監視対象URLの**唯一のマスタ**である。
- URL は**変更しない**。記憶・推測・訓練データからURLを生成・修正することを禁止する。
- 404 / 405 等が返っても、URLを書き換えてはならない。「要確認」として報告するに留める。

## URL 変更の唯一の手順
URLの追加・修正・削除は、以下の **(a) と (b) の両方** を経た変更案を作成し、
**ユーザーの明示承認を得てからのみ** `urls.yaml` に適用する。

- (a) `Invoke-WebRequest`（pwsh）で対象URLのHTTPステータスコードを**実測**する。
- (b) `WebSearch` / `WebFetch` で現行の正しいURLを**確認**する。

(a) または (b) の一方だけ、あるいは承認なしでの適用は禁止。

## フィルタ定義
- 価格・面積の閾値や関心/注意キーワードは `urls.yaml` の `filters:` ブロックを唯一の定義元とする。
- これらの値を `watch.py` 等のコードに直接書かない（ハードコード禁止）。

## 生成物の信頼性
- `data/snapshots/` と `reports/` の内容は `watch.py` の実行結果であり、
  URLが正しいことが前提。URLの正しさを生成物から逆算して判断しない。

## やらないこと（現フェーズ）
- タスクスケジューラへの自動登録はまだ行わない。
